Class: Command

command~Command(name, loaded, config, lang)

The Command class serves as a framework class whose sole purpose is to permit extension by more specific command classes. The most important member is the unimplemented method, Command#execute, which all extending classes are required to implement. This method is called whenever a command is invoked in a server channel, and contains the application logic responsible for processing the implementation of that particular command. The other methods and members belonging to the Command class are the various private properties and their setter/getter methods. Other extending classes may implement their own private prototype methods as needed, but Command exists solely for subsequent extension and thus lacks its own dedicated functionality.

Constructor

new Command(name, loaded, config, lang)

The Command constructor is used simply to store the string, boolean, and Object parameter properties passed during the class's initial initialization (via loadCommand) in private fields via dedicated setters that ensure parameters are of the correct data type.
Parameters:
Name Type Description
name string Name of the command (verify, about, etc.), not the specific configurable text the user must use to invoke the command.
loaded boolean Flag denoting the command's loaded or unloaded status, depending on whether a class instance already exists in the Commander instance's Discord.Collection map, namely Commander#commands.
config Object Ultimately, a reference to the original Client#config Object that is likewise passed to each extension and command on initialization. This parameter contains all the custom configuration options available for tweaking prior to installation.
lang Object A reference to a command-specific property of the Commander plugin's lang property, which is itself a property of the Client class's own lang Object specific to the Commander extension.

execute(message, args, logReply) → {void}

The default execute function is left unimplemented by default in the Command superclass, with the expectation that extending subclasses be the ones to implement this function. It is the most important method of the respective classes, as it is the method invoked when users themselves invoke a command in the Discord server. It coordinates the application logic of the specific command, and may invoke other subclass-specific helper methods as required.
Parameters:
Name Type Description
message Object A new Discord.Message class instance containing information pertaining to the most recent server message, its author, and the channel in which it was posted, among other data.
args Array.<string> An array of strings constituting the arguments that follow the initial command invocation. In the case of verifications and "!about" queries, these arguments are the words that make up the username as separated by spaces; for example, the username "Jabba the Hutt" would be passed as ["Jabba", "the", "Hutt"] as the args argument.
logReply function A reference to the Commander extension's own copy of Extension#addReply method, bound to that extension's class instance. This ensures command subclasses are not directly interfacing with the server channels, a responsibility under the exclusive purview of the Commander extension class and its respective methods alone.
