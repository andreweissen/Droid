Class: Commander

commander~Commander()

The Commander class is a subclass implementation of the base Extension superclass that serves to coordinate the loading and initialization of the Droid bot application's command functionality. This plugin is used by the server's members to interact with the bot in various text channels, with associated functionality ranging from the handling of verification requests to the processing of users' requests for information pertaining to a given user on the wiki.

As with all subclasses extending Extension, the Commander class includes an implementation of the required Extension#onMessage method. This method serves as the event listener callback function that is attached to "message" events by the Client and invoked when a new message is added to a server channel. This particular implementation of the method checks if the new message includes a command invocation, and handles the request accordingly if it does.

commands :Discord.Collection

The commands property of the Commander class is an instance of the Discord.Collection class that is used to store initialized instances of command classes that extend the base Command class. A subclass that extends the base Map JavaScript structure, it is populated by loadCommandDir, usually at the start of the extension's initialization, though post-initialization additions of command classes is possible.

Methods

loadCommand(file, diropt) → {void}

As its name implies, loadCommand is used to create a new class instance of the given command class specified in the file parameter and add that newly created instance to the Commander instance's Discord.Collection map Commander#commands for subsequent retrieval and usage. It is primarily invoked within the context of loadCommandDir during the extension's initialization, though it may be invoked independently by onMessage if the command has somehow not been loaded previously.

loadCommandDir(diropt) → {void}

The loadCommandDir method is used to fetch the string names representing files stored in the parameter directory, filter out those that are not properly suffixed "js" JavaScript modules, and invoke Commander#loadCommand on those for the purposes of populating the Commander extension class instance's Discord.Collection map Commander#commands with commands. This function is invoked by the class's constructor.

onMessage(message) → {void}

Arguably the most important method of the class, the Commander extension's implementation of the required Extension#onMessage method serves as the primary event listener callback function tasked with handling Discord "message" events. As such, it is responsible for checking to see if the bot application should take an interest in the most recent message, namely the Discord.Message instance that is passed as the method's sole parameter. If the message is determined to constitute a command invocation, the method will check to see if a command matching the invoked name exists in Commander#commands, the Discord.Collection containing all previously instantiated command subclasses corresponding to available command functionality. If a match is found, the method invokes the relevant implementation of Command#execute to pass the execution progression off to the relevant subclass.

On a more detailed level, the method begins by checking if the message was a bot post or if the message was not prefixed with the command prefix used to denote a command invocation. In such cases, the method ignores the post. However, if the message does constitute a wellformed command invocation, the method will validate the input and attempt to locate an available command matching the desired command. If one is found, the method will then query Commander#commands to see if an instance of the appropriate command subclass extending Command exists, invoking Commander#loadCommand if not. Execution is then handed off to the Command#execute method to address the required command action.

message Object A new Discord.Message class instance containing information pertaining to the most recent server message, its author, and the channel in which it was posted, among other data.
